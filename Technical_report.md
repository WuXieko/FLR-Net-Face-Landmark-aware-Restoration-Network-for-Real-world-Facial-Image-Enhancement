## 1. 研究背景与问题定义

### 1.1 什么是人脸盲复原

人脸盲复原（Blind Face Restoration）是指：给定一张经历了**未知退化**的低质量人脸图片，将其恢复为高质量的清晰图像。

这里的"盲"（Blind）是关键——在真实场景中，我们不知道照片经历了什么样的损伤。一张从微信收到的模糊人脸照片，可能同时经历了以下退化：

- **光学模糊**：手抖、对焦不准
- **传感器噪声**：高 ISO、暗光环境
- **分辨率损失**：裁剪放大、远距离拍摄
- **压缩伪影**：JPEG 压缩、社交平台二次压缩

模型必须在不知道具体退化类型和程度的情况下，自行判断并修复。

### 1.2 数学描述

退化过程可以抽象为：

```
y = D(x) + n
```

其中：
- `x` 为原始高质量图像（Ground Truth, GT）
- `D(·)` 为未知的退化函数（模糊、下采样、压缩等的组合）
- `n` 为加性噪声
- `y` 为观测到的低质量图像

我们的目标是学习一个复原函数 `F(·)`，使得：

```
F(y) ≈ x
```

即给定退化图 `y`，通过神经网络 `F` 恢复出尽可能接近原图 `x` 的结果。

### 1.3 应用场景

- 老照片修复与翻新
- 监控视频人脸增强
- 社交媒体传输后的画质恢复
- 证件照修复
- 视频会议低画质增强

## 2. 主流方法对比：U-Net / Transformer / NAFNet

图像复原领域有三类主流架构，以下从机理层面逐一分析。

### 2.1 U-Net（经典 CNN 架构）

**核心思想**：编码器-解码器结构 + Skip Connection。

```
输入 → [编码器：逐步下采样] → 瓶颈层 → [解码器：逐步上采样] → 输出
              ↓                                    ↑
              └──────── Skip Connection ────────────┘
```

**机理说明**：
- **编码器**：通过卷积 + 下采样，逐层提取从低级（边缘、纹理）到高级（语义、结构）的特征。每次下采样分辨率减半，通道数翻倍。
- **解码器**：通过上采样 + 卷积，逐层恢复空间分辨率，将高级语义映射回像素空间。
- **Skip Connection**：将编码器每一层的特征直接传递给解码器对应层。因为下采样会丢失空间细节（比如一颗痣的位置），Skip Connection 帮助解码器"记住"这些细节。

**优势**：结构简洁、训练稳定、计算开销适中。

**局限**：
- 卷积的**感受野有限**（3×3 卷积只看到 3×3 的局部区域），对全局信息的捕捉不够。
- 需要通过堆叠很多层才能扩大感受野，导致模型冗余。

### 2.2 Transformer（自注意力架构）

以 SwinIR（Swin Transformer for Image Restoration）为代表。

**核心思想**：用自注意力机制（Self-Attention）替代卷积，让每个像素能直接"看到"图像中所有其他像素。

**自注意力机制原理**：

```
对于图像中的每个位置 i：
  1. 计算 Query(i), Key(i), Value(i)
  2. 用 Query(i) 和所有位置的 Key 做点积 → 得到注意力权重
  3. 用注意力权重加权求和所有位置的 Value → 得到输出
```

通俗来说：每个像素会"询问"其他所有像素"你和我有多大关系？"，关系大的像素对当前像素的输出贡献更大。

**Swin Transformer 的改进**：
- 标准 Self-Attention 的计算复杂度是 O(N²)，N 为像素数量。对于 256×256 的图像，N = 65536，直接计算不可行。
- Swin Transformer 引入**窗口注意力**（Window Attention）：把图像分成小窗口（如 8×8），只在窗口内做注意力，复杂度降为 O(N × M²)，M 为窗口大小。
- 通过**移位窗口**（Shifted Window）机制，让相邻窗口之间也能交换信息。

**优势**：全局建模能力强，对大范围退化（如大面积模糊）效果好。

**局限**：
- **计算量大**：即使有窗口优化，仍然比 CNN 慢 2~5 倍。
- **训练数据需求大**：Transformer 参数多，小数据集容易过拟合。
- **实现复杂**：注意力掩码、位置编码等工程细节多，调试困难。
- **对低级特征捕捉不足**：Transformer 擅长全局关系，但对局部纹理细节的捕捉不如 CNN。

### 2.3 NAFNet（本项目采用的架构）

**核心思想**：不用非线性激活函数（Nonlinear Activation Free），用更简单的操作达到更好的效果。

NAFNet 的设计哲学是"Simple Baselines"——回到 CNN 基础，用最简单的模块搭出强效果。它的关键创新是 **SimpleGate** 机制：

```
传统方法：  x → Conv → ReLU(x) → Conv    （用 ReLU 引入非线性）
NAFNet：    x → Conv → [x1, x2] → x1 * x2 → Conv  （用通道相乘引入非线性）
```

**SimpleGate 机理**：
- 将特征沿通道维度分成两半：`x1, x2 = split(x)`
- 输出 = `x1 × x2`（逐元素相乘）
- 这个简单的乘法操作本身就是非线性的（两个变量的乘积是非线性函数），不需要额外的激活函数
- 实验证明效果优于 ReLU、GELU 等传统激活函数

### 2.4 三种架构定量对比

以下对比基于图像去模糊任务（GoPro 数据集）的公开基准：

| 架构 | 代表模型 | 参数量 | PSNR (dB) | 推理速度 | 训练难度 |
|------|---------|--------|-----------|---------|---------|
| U-Net | MPRNet | 20.1M | 32.66 | 中 | 低 |
| Transformer | Restormer | 26.1M | 32.92 | 慢 | 高 |
| **NAFNet** | **NAFNet-32** | **17.1M** | **32.87** | **快** | **低** |

**结论**：NAFNet 以最少的参数量和最简单的结构，达到了与 Transformer 相当的性能，同时推理速度更快、训练更稳定。这对于本科毕设的硬件条件（单张 RTX 5070 Laptop GPU）来说是最佳选择。

**选择 NAFNet 的理由**：
1. **性能优秀**：接近 SOTA，超越大部分 CNN 方法
2. **训练友好**：不需要复杂的训练技巧，收敛稳定
3. **资源节约**：参数量小，单 GPU 即可训练
4. **代码简洁**：核心组件少，便于理解和修改

## 3. NAFNet 模型架构详解

### 3.1 整体结构

NAFNet 采用 U-Net 式的编码器-解码器结构：

```
输入图像 [B, 3, 256, 256]
    │
    ├── Intro Conv (3→32 通道)
    │
    ├── Encoder Stage 1: 1× NAFBlock, channels=32
    │   └── Downsample (32→64, 分辨率 /2)
    │
    ├── Encoder Stage 2: 1× NAFBlock, channels=64
    │   └── Downsample (64→128, 分辨率 /2)
    │
    ├── Encoder Stage 3: 1× NAFBlock, channels=128
    │   └── Downsample (128→256, 分辨率 /2)
    │
    ├── Encoder Stage 4: 28× NAFBlock, channels=256  ← 主要计算集中在这里
    │   └── Downsample (256→512, 分辨率 /2)
    │
    ├── Middle: 1× NAFBlock, channels=512
    │
    ├── Decoder Stage 1: PixelShuffle 上采样 (512→256)
    │   └── 1× NAFBlock + Skip Connection
    │
    ├── Decoder Stage 2: PixelShuffle 上采样 (256→128)
    │   └── 1× NAFBlock + Skip Connection
    │
    ├── Decoder Stage 3: PixelShuffle 上采样 (128→64)
    │   └── 1× NAFBlock + Skip Connection
    │
    ├── Decoder Stage 4: PixelShuffle 上采样 (64→32)
    │   └── 1× NAFBlock + Skip Connection
    │
    └── Ending Conv (32→3 通道)
         │
         输出图像 [B, 3, 256, 256]
```

**设计要点**：
- 编码器第 4 阶段堆了 28 个 NAFBlock，占总参数量的绝大部分。这是因为在最低分辨率（16×16）上做大量计算，既能捕捉全局信息，又不会太耗显存。
- 解码器每层只用 1 个 NAFBlock，因为有 Skip Connection 提供细节信息，不需要太多计算。
- 下采样用 2×2 步长卷积（不是池化），上采样用 PixelShuffle（比反卷积更平滑，不会产生棋盘伪影）。

### 3.2 NAFBlock 详解

NAFBlock 是 NAFNet 的核心构建单元，每个 block 包含两个分支：

```
输入 x
 │
 ├── 分支 1：空间注意力
 │   ├── LayerNorm
 │   ├── 1×1 Conv (升维: c → 2c)
 │   ├── 3×3 Depthwise Conv (空间信息提取)
 │   ├── SimpleGate (2c → c)
 │   ├── SCA: Simplified Channel Attention
 │   │   ├── Global Average Pooling
 │   │   └── 1×1 Conv (学习通道权重)
 │   ├── 1×1 Conv (降维)
 │   └── × beta (可学习缩放系数)
 │
 ├── 残差连接: y = x + 分支1输出
 │
 ├── 分支 2：FFN (Feed-Forward Network)
 │   ├── LayerNorm
 │   ├── 1×1 Conv (升维: c → 2c)
 │   ├── SimpleGate (2c → c)
 │   ├── 1×1 Conv (降维)
 │   └── × gamma (可学习缩放系数)
 │
 └── 残差连接: output = y + 分支2输出
```

**关键组件说明**：

**SimpleGate**：
```python
def forward(self, x):
    x1, x2 = x.chunk(2, dim=1)  # 通道分两半
    return x1 * x2               # 逐元素相乘
```
看似简单，但 x1 × x2 本身就是非线性操作。一半通道起"选择"作用（类似门控），另一半通道提供"内容"。

**SCA (Simplified Channel Attention)**：
```python
# 全局平均池化 → 1×1 卷积 → 得到每个通道的权重
attention = AdaptiveAvgPool2d(1)(x)  # [B, C, 1, 1]
attention = Conv2d(C, C, 1)(attention)
output = x * attention  # 通道加权
```
让网络自动学习"哪些通道的特征更重要"，增强有用特征、抑制无用特征。

**可学习缩放系数 (beta, gamma)**：
初始化为 0，让网络在训练初期等同于恒等映射（输入 = 输出），避免一开始就产生大的梯度波动，显著提升训练稳定性。

### 3.3 本项目的模型配置

```python
model = NAFNet(
    img_channel=3,           # RGB
    width=32,                # 基础通道数
    middle_blk_num=1,        # 中间层 1 个 NAFBlock

    dec_blk_nums=[1,1,1,1],  # 解码器块数分配
)
# 总参数量: 17.11M

```

## 4. 在线盲退化 Pipeline

### 4.1 设计思路

传统方法是预先生成退化数据（离线退化），但这限制了退化的多样性。本项目采用**在线退化**策略：

```
每次读取一张图时：
  原图 ──→ [随机退化组合] ──→ 退化图
  │                              │
  作为 GT (目标)            作为模型输入 (LQ)
```

每个 epoch 每张图的退化都不同，模型在 80 个 epoch 内见到的退化组合约为 80 × 52001 ≈ 416 万种，远超离线退化的多样性。

### 4.2 四种退化操作

#### 4.2.1 高斯模糊（概率: 70%）

**模拟场景**：对焦不准、手抖运动模糊。

**原理**：用高斯核与图像做卷积。核越大、sigma 越大，模糊越严重。

```python
kernel_size = random.choice([7, 9, 11, 13, 15, 17, 19, 21])
sigma = random.uniform(0.5, 8.0)
img = cv2.GaussianBlur(img, (kernel_size, kernel_size), sigma)
```

参数范围说明：
- kernel_size 7~21：覆盖从轻微到严重的模糊
- sigma 0.5~8.0：控制模糊扩散程度

#### 4.2.2 高斯噪声（概率: 50%）

**模拟场景**：高 ISO 拍摄、暗光环境、老照片颗粒。

**原理**：在每个像素值上叠加均值为 0、标准差为 σ 的高斯随机数。

```python
noise_level = random.uniform(1, 50) / 255.0
noise = np.random.normal(0, noise_level, img.shape)
img = np.clip(img + noise, 0, 1)
```

参数范围说明：
- noise_level 1~50/255：覆盖从轻微到严重的噪声
- clip 到 [0,1]：防止像素值越界

#### 4.2.3 下采样 + 上采样（概率: 40%）

**模拟场景**：低分辨率图片放大、远距离拍摄裁剪。

**原理**：先缩小图片再放大回原尺寸，高频细节在缩小时被丢弃，放大时无法恢复。

```python
scale = random.uniform(0.3, 0.9)
h, w = img.shape[:2]
small = cv2.resize(img, (int(w*scale), int(h*scale)))          # 缩小
img = cv2.resize(small, (w, h), interpolation=cv2.INTER_LINEAR) # 放大
```

参数范围说明：
- scale 0.3~0.9：最多缩小到原来的 30%，相当于 3.3 倍超分

#### 4.2.4 JPEG 压缩（概率: 60%）

**模拟场景**：微信/QQ 传图、网页截图、社交平台压缩。

**原理**：JPEG 编码会在 DCT（离散余弦变换）域做量化，quality 越低量化越粗糙，高频信息丢失越多，产生块状伪影。

```python
quality = random.randint(30, 90)
_, encoded = cv2.imencode('.jpg', img_uint8, [cv2.IMWRITE_JPEG_QUALITY, quality])
img = cv2.imdecode(encoded, cv2.IMREAD_COLOR)
```

参数范围说明：
- quality 30~90：30 为严重压缩（明显块效应），90 为轻微压缩

### 4.3 退化组合策略

四种退化的施加是**独立概率的级联**（不是互斥选择），这意味着：

- 一张图可能同时经历 2~4 种退化（概率最高的情况）
- 也可能只经历 1 种退化
- 极端情况下可能全部命中或全部未命中（已加兜底机制）

这种设计使得退化分布更接近真实世界的情况——一张微信图片可能同时被模糊、加噪、压缩。

## 5. 损失函数设计

### 5.1 为什么需要多个损失函数

单一的 L1 Loss 虽然能保证像素级准确性，但会产生**过度平滑**的结果——因为对于有多个合理解的位置（如头发丝），取平均是 L1 的最优解，而平均值恰恰是模糊的。

因此我们需要多个损失函数从不同维度约束模型输出。

### 5.2 L1 Loss（像素级损失）

**作用**：保证输出图像在像素层面与 GT 尽可能接近。

**公式**：

```
L_L1 = (1/N) × Σ |F(y) - x|
```

**特点**：
- 对异常值不敏感（相比 L2 Loss）
- 是训练的"主力"，权重 = 1.0
- 保证整体结构和颜色正确

### 5.3 Perceptual Loss（感知损失）

**作用**：让输出在人类视觉感知上与 GT 相似，即"看起来像"。

**原理**：利用预训练 VGG19 网络提取特征，比较特征空间的距离而非像素空间。

```
L_perc = (1/N) × Σ |VGG(F(y)) - VGG(x)|
```

**为什么有效**：
- VGG 是在 ImageNet 上训练的分类网络，它的中间层特征编码了人类视觉系统关注的语义信息
- 两张图在 VGG 特征空间距离小 = 人类觉得它们看起来相似
- 避免了 L1 Loss 的过度平滑问题

**实现细节**：
```python
vgg = models.vgg19(pretrained=True).features[:16]  # 只取前 16 层 (到 relu3_3)
# 浅层特征: 边缘、纹理
# 中层特征: 局部结构、纹理模式
# 我们取到中层，平衡底层细节和高层语义
```

权重 = 0.1（太大会产生不自然的纹理伪影）

### 5.4 FFT Loss（频域损失）

**作用**：直接在频域惩罚高频细节的缺失，恢复图像锐度。

**原理**：对图像做 2D 傅里叶变换，将图像从空间域转换到频率域，然后比较幅度谱。

```
L_FFT = (1/N) × Σ ||FFT(F(y))| - |FFT(x)||
```

**为什么有效**：
- 图像的高频分量 = 边缘、纹理、细节
- 退化过程（模糊、压缩）主要破坏高频信息
- 在频域直接约束高频分量，比在空间域间接约束更直接有效

**实现细节**：
```python
pred_fft = torch.fft.fft2(pred.float(), norm='ortho')   # 2D FFT
target_fft = torch.fft.fft2(target.float(), norm='ortho')
loss = F.l1_loss(torch.abs(pred_fft), torch.abs(target_fft))  # 只比较幅度
```

权重 = 0.05（太大会产生伪纹理）

### 5.5 组合损失总公式

```
L_total = 1.0 × L_L1 + 0.1 × L_perc + 0.05 × L_FFT
```

各分量的角色分工：
- **L1**："基本面对不对" → 保证像素级准确
- **Perceptual**："看起来像不像" → 保证感知质量
- **FFT**："细节清不清楚" → 保证高频锐度

## 6. 训练策略与实现

### 6.1 训练数据

| 数据集 | 图片数量 | 分辨率 | 用途 |
|--------|---------|--------|------|
| FFHQ | 52,001 | 512×512 | 正式训练（高质量对齐人脸） |
| WIDER FACE | 12,880 | 混合 | 预验证流程（复杂场景人脸检测数据） |

**为什么选 FFHQ**：
- Flickr-Faces-HQ 是人脸生成/复原领域的标准数据集
- 所有图片都经过对齐和裁剪，质量一致
- 覆盖不同年龄、性别、种族、光照条件
- 512×512 分辨率足够模型学习精细特征

### 6.2 训练超参数

| 参数 | 值 | 说明 |
|------|----|------|
| Batch Size | 8 | RTX 5070 Laptop 下的最优配置 |
| Patch Size | 256×256 | 从 512×512 原图中随机裁剪 |
| Learning Rate | 2×10⁻⁴ | AdamW 初始学习率 |
| Weight Decay | 1×10⁻⁴ | 正则化防止过拟合 |
| Epochs | 80 | FFHQ 正式训练 |
| LR Scheduler | CosineAnnealing | 余弦退火，eta_min=1×10⁻⁶ |
| Mixed Precision | bfloat16 | 节省约 30% 显存 |
| Gradient Clipping | max_norm=1.0 | 防止梯度爆炸 |

### 6.3 训练流程

```
每个 Epoch:
  ┌─────────────────────────────────────────────────┐
  │ 1. 从 FFHQ 读取一张 512×512 原图                  │
  │ 2. 随机裁剪 256×256 的 patch                      │
  │ 3. 随机翻转（水平/垂直）做数据增强                    │
  │ 4. 用 BlindDegradation 在线生成退化版本              │
  │ 5. 退化图 → NAFNet → 复原图                       │
  │ 6. 计算 CombinedLoss(复原图, 原图)                 │
  │ 7. 反向传播 + 梯度裁剪 + 参数更新                    │
  │ 8. 每 50 step 记录 TensorBoard                    │
  │ 9. 每 500 step 保存对比图                          │
  │ 10. 每 5 epoch 保存 checkpoint                     │
  └─────────────────────────────────────────────────┘
```

### 6.4 混合精度训练

使用 bfloat16 混合精度，原理是：

- 前向传播和反向传播中，大部分计算用 bfloat16（16位浮点，省显存，计算快）
- 权重更新用 float32（保证精度，避免数值溢出）
- GradScaler 自动处理损失缩放，防止 bfloat16 下梯度下溢

**效果**：显存占用降低约 30%，训练速度提升约 20%，精度几乎无损。

### 6.5 断点续训

训练脚本支持从 checkpoint 恢复，每 5 个 epoch 自动保存一次：
```bash
# 首次训练
python train.py

# 中断后恢复（自动找到最新 checkpoint）
python train.py --resume

# 指定 checkpoint 恢复
python train.py --resume --ckpt checkpoints/epoch_30.pth
```

恢复时会加载：模型权重、优化器状态、学习率调度器状态、当前 epoch 数。

### 6.6 实际训练记录

| 阶段 | 数据集 | Epochs | 初始 Loss | 最终 Loss | 耗时 |
|------|--------|--------|-----------|-----------|------|
| 预验证 | WIDER FACE | 30 | 0.75 | 0.10 | ~10h |
| 正式训练 | FFHQ | 80 | 0.13 | 0.08 | ~22h |

训练曲线显示 Loss 从 0.13 稳步下降到 0.07~0.08，无异常震荡或爆炸，收敛正常。

## 7. 实验结果与分析

### 7.1 实验设置

- **测试集**：从 FFHQ 后 10%（5,201 张）中随机抽取 200 张
- **退化方式**：与训练时相同的在线盲退化
- **评价指标**：PSNR（峰值信噪比）、SSIM（结构相似性）
- **随机种子**：固定为 42，保证实验可复现

### 7.2 实验 1：测试集定量评估

在 200 张测试图片上的平均结果：

| Metric | Degraded | Restored | Gain |
|--------|----------|----------|------|
| PSNR (dB) | 26.85 | **31.98** | **+5.13** |
| SSIM | 0.9576 | **0.9913** | **+0.0337** |

**分析**：
- 复原后 PSNR 达到 31.98 dB，超过 30 dB 的"优秀"基准线
- SSIM 达到 0.9913，接近 1.0（完美还原），说明结构信息几乎完全恢复
- 平均 PSNR 提升 5.13 dB，在盲复原任务中属于显著改善

### 7.3 实验 2：单项退化分析

分别只施加一种退化类型，测试模型对每种退化的修复能力：

| Degradation Type | PSNR Degraded | PSNR Restored | Gain |
|------------------|---------------|---------------|------|
| Gaussian Noise | 19.12 | 32.76 | **+13.65** |
| Gaussian Blur | 26.08 | 30.99 | +4.92 |
| Combined | 25.65 | 30.32 | +4.68 |
| JPEG Compress | 33.13 | 34.52 | +1.39 |
| Downsample | 29.02 | 28.76 | -0.27 |

**分析**：

- **去噪能力最强**（+13.65 dB）：高斯噪声是最严重的退化（PSNR 仅 19.12），模型将其恢复到 32.76 dB，提升幅度巨大。这说明 NAFNet 对噪声模式有很好的建模能力。

- **去模糊效果良好**（+4.92 dB）：模糊信息虽然不可逆（高频信息被卷积核"抹平"），但模型通过学习人脸先验知识，能在一定程度上"想象"回缺失的细节。

- **组合退化有效**（+4.68 dB）：面对多种退化叠加的复杂场景，模型仍能有效修复，验证了盲复原的能力。

- **JPEG 压缩提升有限**（+1.39 dB）：因为 JPEG quality=40 的退化本身较轻（33.13 dB），提升空间小。

- **下采样略有下降**（-0.27 dB）：4 倍下采样后信息丢失极为严重，这已经属于超分辨率任务的范畴，NAFNet 并非专门为此设计。这是模型的**已知局限**，后续可通过引入超分辨率模块或增大退化 pipeline 中下采样的概率来改善。

### 7.4 实验 3：视觉对比

推理脚本会生成三图对比（原图 | 退化图 | 复原图）并在底部标注 PSNR 和 SSIM。精选样例的 PSNR 提升范围为 +1.84 ~ +15.55 dB，展示了模型在不同退化程度下的修复能力。

### 7.5 训练曲线

通过 TensorBoard 记录的 Loss 曲线显示：

- **Loss/total**：从 0.13 稳步下降到 0.07~0.08，呈现典型的先快后慢收敛趋势
- **Loss/l1**：主力损失持续下降，说明像素级还原能力不断提升
- **Loss/perceptual**：在训练前期快速下降，后期趋于平稳，说明语义层面的学习已基本完成
- **Loss/fft**：波动较大但整体呈下降趋势，高频细节的恢复在持续改进

### 8.2 推理脚本 test.py —— 复原图片并生成对比图

`test.py` 的作用是：接收一张或一批图片 → 用训练好的退化 Pipeline 对图片进行退化 → 用模型复原 → 生成三图对比图（原图 | 退化图 | 复原图）并在底部标注 PSNR。

#### 基本用法

```bash
# 1. 复原单张图片（使用最优模型 best.pth）
python test.py --input data/ffhq/00001.png

# 2. 复原单张图片（指定其他 checkpoint）
python test.py --input data/ffhq/00001.png --ckpt checkpoints/epoch_80.pth

# 3. 复原整个文件夹的所有图片
python test.py --input data/ffhq/

# 4. 复原图片并指定输出目录
python test.py --input my_blurry_face.jpg --output my_results
```

#### 完整参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--input` | (必填) | 输入图片路径或文件夹路径 |
| `--ckpt` | `checkpoints/best.pth` | 模型权重路径 |
| `--output` | `results` | 输出目录 |
| `--width` | `32` | NAFNet 通道数，必须和训练时一致 |

#### 输出内容

运行后在输出目录中生成：

| 文件 | 内容 |
|------|------|
| `{filename}_compare.png` | 三图对比大图，底部标注红色（退化 PSNR）、绿色（复原 PSNR）、黄色（PSNR 提升） |
| `{filename}_restored.png` | 单独的复原结果图 |

终端同时输出每张图的 PSNR 指标和最终的平均汇总：

```
============================================================
✅ 全部完成！共处理 1 张图片

   平均退化 PSNR:  28.18 dB
   平均复原 PSNR:  31.62 dB

```

## 9. 总结与后续工作

### 9.1 工作总结

本项目实现了一个基于 NAFNet 的人脸盲复原系统，核心贡献包括：

1. **模型选择与实现**：选用 NAFNet 作为骨干网络，实现了完全自包含的模型代码（不依赖外部库），参数量 17.11M，在单 GPU 上即可高效训练。

2. **在线盲退化 Pipeline**：设计了包含高斯模糊、高斯噪声、下采样、JPEG 压缩四种退化的在线随机退化 pipeline，使模型能泛化到真实世界的未知退化场景。

3. **多维度损失函数**：组合 L1、Perceptual、FFT 三种损失函数，分别从像素、语义、频域三个层面约束模型输出，避免了单一 L1 Loss 的过度平滑问题。

4. **系统性实验验证**：在 FFHQ 数据集上进行了测试集定量评估、单项退化分析、视觉对比三组实验，验证了系统的有效性。

### 9.2 实验结论

- 模型在 200 张测试图片上达到平均 PSNR 31.98 dB、SSIM 0.9913 的复原质量
- 对高斯噪声的修复效果最为突出（+13.65 dB）
- 对组合退化的处理能力良好（+4.68 dB），验证了盲复原能力
- 对严重下采样（4×）的修复能力有限，属于已知局限

### 9.3 后续改进方向

1. **增大模型容量**：将 width 从 32 提升到 64（参数量约 17M → 68M），预期可进一步提升 PSNR 1~2 dB。

2. **引入人脸先验**：利用人脸关键点、人脸解析图（Face Parsing Map）等先验信息，引导模型更好地恢复五官细节。

3. **增加超分辨率模块**：针对下采样退化的局限，可在 NAFNet 后接一个轻量级超分辨率头，提升对低分辨率输入的处理能力。

4. **GAN 训练**：引入对抗训练（Adversarial Training），使用判别器约束生成图像的真实性，进一步提升视觉感知质量。

5. **消融实验**：系统性地分析每个损失函数分量的贡献，验证设计选择的合理性。
